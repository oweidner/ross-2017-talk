Second Generation HPC Applications

  • Diverging from tightly-coupled monoliths ... (multi-component workflows, many-task, etc.)
  • Don't (entirely) fit into the narrow HPC usage model
  • Not always compute-centric ... (e.g., Big-Data)
  • Applications that come from a non-HPC context
  • Distributed, leverage multiple platforms and federations
  • Traditionalists claim: applications HPC systems weren't designed for ...
  • Depending who you talk to

Assumption 1

All data that is observable on an HPC platform without creating significant overhead should be observed and made available with the shortes possible delay through public interfaces to users, developers and platform operators alike.

Assumption 2

If large amount of data is produced, guiding systems and principles are required that help to extract and distill relevant insights from the data efficiently.

Assumption 3

Assuming that platforms and applications continue to grow and with it the amount of data generated, scalability is the main concern for any data-driven HPC platform.

Assumption 4

Different user and application groups require data at different levels, from raw data to high-level, derived signals.

Assumption 5

Introducing a operational data ecosystem on existing HPC platforms can only be succesfull if it is non-disruptive and minimally invasive.
